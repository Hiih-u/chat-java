# 测试运行指南

## 测试概述

项目包含两类测试：
- **单元测试**：[`ConversationServiceTest.java`](../src/test/java/com/ai/chat/service/ConversationServiceTest.java) - 测试业务逻辑层
- **集成测试**：[`ConversationControllerTest.java`](../src/test/java/com/ai/chat/controller/ConversationControllerTest.java) - 测试 Controller 层

测试使用 **JUnit 5** + **Mockito** + **MockMvc** 框架，使用 **H2 内存数据库**，无需配置真实数据库。

---

## 方式一：使用 Maven 命令行

### 1. 运行所有测试

```bash
cd chat-java/chat-system
mvn test
```

### 2. 运行指定测试类

```bash
# 只运行 Service 层测试
mvn test -Dtest=ConversationServiceTest

# 只运行 Controller 层测试
mvn test -Dtest=ConversationControllerTest
```

### 3. 运行指定测试方法

```bash
# 运行单个测试方法
mvn test -Dtest=ConversationServiceTest#testCreateConversation_Success
```

### 4. 跳过测试（打包时）

```bash
mvn clean package -DskipTests
```

### 5. 生成测试报告

```bash
mvn test
# 报告位置：target/surefire-reports/
```

## 方式二：使用 IDE（推荐）

### 在 IntelliJ IDEA 中运行

1. **运行所有测试**
   - 右键点击 `src/test/java` 目录
   - 选择 `Run 'All Tests'`

2. **运行单个测试类**
   - 打开测试类文件（如 [`ConversationServiceTest.java`](../src/test/java/com/ai/chat/service/ConversationServiceTest.java)）
   - 点击类名旁边的绿色运行按钮
   - 或右键类名 → `Run 'ConversationServiceTest'`

3. **运行单个测试方法**
   - 点击测试方法旁边的绿色运行按钮
   - 或右键方法名 → `Run 'testCreateConversation_Success()'`

4. **调试测试**
   - 点击绿色按钮旁边的 Debug 图标
   - 或右键 → `Debug 'ConversationServiceTest'`

### 在 VS Code 中运行

1. **安装插件**
   - 安装 `Extension Pack for Java`（包含测试支持）
   - 安装 `Test Runner for Java`

2. **运行测试**
   - 打开测试文件，会看到每个测试方法上方有 `Run Test` 链接
   - 点击即可运行单个测试
   - 或在测试资源管理器中运行

## 测试配置说明

### 测试配置文件

[`application-test.yml`](../src/test/resources/application-test.yml) 配置了测试环境：

```yaml
spring:
  datasource:
    driver-class-name: org.h2.Driver
    url: jdbc:h2:mem:testdb;MODE=PostgreSQL
    username: sa
    password:
```

- 使用 **H2 内存数据库**，模拟 PostgreSQL 模式
- 每次测试运行时自动创建和销毁数据库
- 无需配置真实数据库连接

### 测试依赖

在 [`pom.xml`](../pom.xml) 中已配置：

```xml
<!-- Spring Boot Test -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>

<!-- Mockito -->
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <scope>test</scope>
</dependency>

<!-- H2 Database for testing -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>test</scope>
</dependency>
```

- ✅ PUT `/api/conversation/{id}` - 更新会话
- ✅ DELETE `/api/conversation/{id}` - 删除会话
- ✅ DELETE `/api/conversation/batch` - 批量删除

## 常见问题

### 1. 测试失败：找不到数据库驱动

**问题**：`java.lang.ClassNotFoundException: org.h2.Driver`

**解决**：确保 [`pom.xml`](../pom.xml) 中有 H2 依赖，然后运行：

```bash
mvn clean install
```

### 2. 测试失败：端口被占用

**问题**：`Port 8080 is already in use`

**解决**：测试使用 `@WebMvcTest`，不会启动真实服务器，不应该有端口冲突。如果有，检查是否有其他测试在运行。

### 3. 测试失败：Bean 注入失败

**问题**：`No qualifying bean of type...`

**解决**：
- 检查 `@MockBean` 注解是否正确
- 确保测试类上有正确的注解（`@WebMvcTest` 或 `@ExtendWith(MockitoExtension.class)`）

### 4. 如何调试测试

在 IDE 中：
1. 在测试代码中设置断点
2. 右键测试方法 → `Debug 'testXxx()'`
3. 程序会在断点处暂停，可以查看变量值

## 最佳实践

### 1. 测试命名规范

```java
@Test
@DisplayName("创建会话 - 成功")
void testCreateConversation_Success() {
    // 测试代码
}
```

- 方法名：`test + 方法名 + 场景`
- DisplayName：中文描述，清晰说明测试内容

### 2. 测试结构（AAA 模式）

```java
@Test
void testExample() {
    // Arrange（准备）- 设置测试数据和 Mock 行为
    when(mapper.selectById(1L)).thenReturn(entity);
    
    // Act（执行）- 调用被测试的方法
    Result result = service.doSomething(1L);
    
    // Assert（断言）- 验证结果
    assertNotNull(result);
    assertEquals(expected, result);
    verify(mapper, times(1)).selectById(1L);
}
```

### 3. 使用 @BeforeEach 准备测试数据

```java
@BeforeEach
void setUp() {
    // 准备每个测试都需要的数据
    testData = new TestData();
}
```

### 4. 测试覆盖率目标

- 业务逻辑层（Service）：**80%+**
- 控制器层（Controller）：**70%+**
- 工具类：**90%+**

## 持续集成（CI）

在 CI/CD 流程中自动运行测试：

### GitHub Actions 示例

```yaml
name: Java CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up JDK 17
        uses: actions/setup-java@v2
        with:
          java-version: '17'
      - name: Run tests
        run: mvn test
```

### GitLab CI 示例

```yaml
test:
  stage: test
  script:
    - mvn test
  artifacts:
    reports:
      junit: target/surefire-reports/TEST-*.xml
```

## 快速开始

**推荐的测试流程**：

1. **首次运行**：在项目根目录执行
   ```bash
   cd chat-java/chat-system
   mvn clean test
   ```

2. **开发时**：在 IDE 中运行单个测试方法，快速验证

3. **提交前**：运行所有测试，确保没有破坏现有功能
   ```bash
   mvn test
   ```

4. **查看报告**：检查 `target/surefire-reports/` 目录
