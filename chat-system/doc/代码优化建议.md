# 代码优化建议与最佳实践

## 一、GlobalExceptionHandler 优化

### 当前问题

[`GlobalExceptionHandler.java`](../chat-common/src/main/java/com/ai/chat/common/handler/GlobalExceptionHandler.java) 存在以下问题：

1. ❌ 缺少 `@ExceptionHandler` 注解 - 方法不会被触发
2. ❌ 异常处理不全面 - 只处理了 BusinessException
3. ❌ 缺少参数校验异常处理
4. ❌ 缺少通用异常兜底处理
5. ❌ 日志记录不完整

### 优化后的代码

```java
package com.ai.chat.common.handler;

import com.ai.chat.common.entity.Result;
import com.ai.chat.common.enums.ResultCode;
import com.ai.chat.common.exception.BusinessException;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.HttpStatus;
import org.springframework.validation.BindException;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ResponseStatus;
import org.springframework.web.bind.annotation.RestControllerAdvice;

import jakarta.validation.ConstraintViolation;
import jakarta.validation.ConstraintViolationException;
import java.util.stream.Collectors;

/**
 * 全局异常处理器
 * 统一处理各类异常，返回标准化的错误响应
 */
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    /**
     * 处理业务异常
     */
    @ExceptionHandler(BusinessException.class)
    @ResponseStatus(HttpStatus.OK)
    public Result<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常: code={}, message={}", e.getCode(), e.getMessage());
        return Result.error(e.getCode(), e.getMessage());
    }

    /**
     * 处理参数校验异常（@Valid 注解）
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Result<Void> handleMethodArgumentNotValidException(MethodArgumentNotValidException e) {
        String errorMsg = e.getBindingResult().getFieldErrors().stream()
                .map(FieldError::getDefaultMessage)
                .collect(Collectors.joining("; "));
        log.warn("参数校验失败: {}", errorMsg);
        return Result.error(ResultCode.BAD_REQUEST, errorMsg);
    }

    /**
     * 处理参数绑定异常
     */
    @ExceptionHandler(BindException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Result<Void> handleBindException(BindException e) {
        String errorMsg = e.getFieldErrors().stream()
                .map(FieldError::getDefaultMessage)
                .collect(Collectors.joining("; "));
        log.warn("参数绑定失败: {}", errorMsg);
        return Result.error(ResultCode.BAD_REQUEST, errorMsg);
    }

    /**
     * 处理约束违反异常（@Validated 注解）
     */
    @ExceptionHandler(ConstraintViolationException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Result<Void> handleConstraintViolationException(ConstraintViolationException e) {
        String errorMsg = e.getConstraintViolations().stream()
                .map(ConstraintViolation::getMessage)
                .collect(Collectors.joining("; "));
        log.warn("约束校验失败: {}", errorMsg);
        return Result.error(ResultCode.BAD_REQUEST, errorMsg);
    }

    /**
     * 处理非法参数异常
     */
    @ExceptionHandler(IllegalArgumentException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public Result<Void> handleIllegalArgumentException(IllegalArgumentException e) {
        log.warn("非法参数: {}", e.getMessage());
        return Result.error(ResultCode.BAD_REQUEST, e.getMessage());
    }

    /**
     * 处理空指针异常
     */
    @ExceptionHandler(NullPointerException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<Void> handleNullPointerException(NullPointerException e) {
        log.error("空指针异常", e);
        return Result.error(ResultCode.INTERNAL_ERROR, "系统内部错误");
    }

    /**
     * 处理所有未捕获的异常（兜底）
     */
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error(ResultCode.INTERNAL_ERROR, "系统内部错误，请联系管理员");
    }
}
```

### 学习要点

1. **@ExceptionHandler 注解**：标记异常处理方法
2. **@ResponseStatus 注解**：设置 HTTP 状态码
3. **异常处理优先级**：从具体到通用
4. **日志分级**：warn 用于业务异常，error 用于系统异常
5. **错误信息脱敏**：系统异常不暴露详细信息

## 二、getDetails 方法优化

### 当前问题

[`ConversationServiceImpl.java`](../chat-system/src/main/java/com/ai/chat/service/impl/ConversationServiceImpl.java:86) 的 `getDetails` 方法：

```java
public ConversationResponse getDetails(String conversationIdOrId) {
    Conversation conversation = this.lambdaQuery()
            .eq(Conversation::getConversationId, conversationIdOrId)
            .one();
    // 如果找不到，尝试用数字 ID 查询
    if (conversation == null && conversationIdOrId != null && conversationIdOrId.matches("\\d+")) {
        conversation = this.getById(Long.parseLong(conversationIdOrId));
    }
    if (conversation == null) {
        throw new BusinessException(ResultCode.NOT_FOUND, "会话不存在");
    }
    return ConversationConverter.toResponse(conversation);
}
```

**问题**：
1. ❌ 职责不单一：一个方法处理两种查询逻辑
2. ❌ 缓存键不明确：conversationId 和 id 混用
3. ❌ 性能问题：可能执行两次数据库查询
4. ❌ 语义不清：参数名 `conversationIdOrId` 含糊

### 优化方案

#### 方案一：拆分为两个方法（推荐）

```java
/**
 * 根据 conversationId 获取会话详情
 */
@Override
@Cacheable(value = CacheConfig.CONVERSATION_CACHE, key = "#conversationId", unless = "#result == null")
public ConversationResponse getByConversationId(String conversationId) {
    Conversation conversation = this.lambdaQuery()
            .eq(Conversation::getConversationId, conversationId)
            .one();
    if (conversation == null) {
        throw new BusinessException(ResultCode.NOT_FOUND, "会话不存在");
    }
    return ConversationConverter.toResponse(conversation);
}

/**
 * 根据数据库 ID 获取会话详情
 */
@Override
@Cacheable(value = CacheConfig.CONVERSATION_CACHE, key = "'id:' + #id", unless = "#result == null")
public ConversationResponse getById(Long id) {
    Conversation conversation = super.getById(id);
    if (conversation == null) {
        throw new BusinessException(ResultCode.NOT_FOUND, "会话不存在");
    }
    return ConversationConverter.toResponse(conversation);
}
```

#### 方案二：保留兼容性但优化逻辑

```java
/**
 * 智能查询：优先按 conversationId 查询，失败则尝试按 ID 查询
 * 注意：此方法为兼容旧接口，新代码建议使用 getByConversationId
 */
@Override
public ConversationResponse getDetails(String conversationIdOrId) {
    // 优先按 conversationId 查询（使用缓存）
    ConversationResponse response = getByConversationId(conversationIdOrId);
    if (response != null) {
        return response;
    }
    
    // 如果是数字，尝试按 ID 查询
    if (conversationIdOrId.matches("\\d+")) {
        return getById(Long.parseLong(conversationIdOrId));
    }
    
    throw new BusinessException(ResultCode.NOT_FOUND, "会话不存在");
}
```

### 学习要点

1. **单一职责原则**：一个方法只做一件事
2. **缓存键设计**：不同查询条件使用不同的缓存键
3. **方法命名**：清晰表达方法意图
4. **性能优化**：避免不必要的数据库查询

```java
package com.ai.chat.common.entity;

import com.ai.chat.common.enums.ResultCode;
import com.fasterxml.jackson.annotation.JsonInclude;
import lombok.Data;

import java.io.Serializable;

/**
 * 统一响应结果封装
 */
@Data
@JsonInclude(JsonInclude.Include.NON_NULL)  // 空值不序列化
public class Result<T> implements Serializable {

    private static final long serialVersionUID = 1L;

    private Integer code;
    private String message;
    private T data;
    private Long timestamp;  // 添加时间戳

    // 私有构造函数，强制使用静态工厂方法
    private Result() {
        this.timestamp = System.currentTimeMillis();
    }

    public static <T> Result<T> success(T data) {
        return build(ResultCode.SUCCESS, data);
    }

    public static <T> Result<T> success() {
        return build(ResultCode.SUCCESS, null);
    }

    public static <T> Result<T> error(String message) {
        return build(ResultCode.INTERNAL_ERROR.getCode(), message, null);
    }

    public static <T> Result<T> error(ResultCode resultCode) {
        return build(resultCode, null);
    }

    public static <T> Result<T> error(ResultCode resultCode, String message) {
        return build(resultCode.getCode(), message, null);
    }

    public static <T> Result<T> error(int code, String message) {
        return build(code, message, null);
    }

    public static <T> Result<T> build(ResultCode resultCode, T data) {
        return build(resultCode.getCode(), resultCode.getMessage(), data);
    }

    public static <T> Result<T> build(int code, String message, T data) {
        Result<T> result = new Result<>();
        result.setCode(code);
        result.setMessage(message);
        result.setData(data);
        return result;
    }

    /**
     * 判断是否成功
     */
    public boolean isSuccess() {
        return ResultCode.SUCCESS.getCode() == this.code;
    }
}
```

### 2. BusinessException 优化

**优化建议**：添加更多构造函数和工厂方法

```java
package com.ai.chat.common.exception;

import com.ai.chat.common.enums.ResultCode;
import lombok.Getter;

@Getter
public class BusinessException extends RuntimeException {

    private final int code;
    private final Object[] args;  // 支持消息参数化

    public BusinessException(String message) {
        super(message);
        this.code = ResultCode.BAD_REQUEST.getCode();
        this.args = null;
    }

    public BusinessException(ResultCode resultCode) {
        super(resultCode.getMessage());
        this.code = resultCode.getCode();
        this.args = null;
    }

    public BusinessException(ResultCode resultCode, String message) {
        super(message);
        this.code = resultCode.getCode();
        this.args = null;
    }

    public BusinessException(ResultCode resultCode, String message, Object... args) {
        super(message);
        this.code = resultCode.getCode();
        this.args = args;
    }

    // 工厂方法
    public static BusinessException notFound(String resource) {
        return new BusinessException(ResultCode.NOT_FOUND, resource + " 不存在");
    }

    public static BusinessException badRequest(String message) {
        return new BusinessException(ResultCode.BAD_REQUEST, message);
    }

    public static BusinessException forbidden(String message) {
        return new BusinessException(ResultCode.FORBIDDEN, message);
    }
}
```

### 3. 添加自定义校验注解

创建 `@ConversationId` 注解：

```java
package com.ai.chat.common.validation;

import jakarta.validation.Constraint;
import jakarta.validation.Payload;
import java.lang.annotation.*;

@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = ConversationIdValidator.class)
@Documented
public @interface ConversationId {
    String message() default "conversationId 格式不正确";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}
```

校验器：

```java
package com.ai.chat.common.validation;

import jakarta.validation.ConstraintValidator;
import jakarta.validation.ConstraintValidatorContext;

public class ConversationIdValidator implements ConstraintValidator<ConversationId, String> {

    private static final String CONVERSATION_ID_PATTERN = "^conv_[a-zA-Z0-9]{6,32}$";

    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null || value.isEmpty()) {
            return false;
        }
        return value.matches(CONVERSATION_ID_PATTERN);
    }
}
```

## 四、chat-system 模块优化建议

### 1. Controller 层优化

**当前代码**：[`ConversationController.java`](../chat-system/src/main/java/com/ai/chat/controller/ConversationController.java)

**优化建议**：

```java
package com.ai.chat.controller;

import com.ai.chat.common.Result;
import com.ai.chat.dto.request.ConversationCreateRequest;
import com.ai.chat.dto.request.ConversationUpdateRequest;
import com.ai.chat.dto.response.ConversationResponse;
import com.ai.chat.service.IConversationService;
import com.baomidou.mybatisplus.extension.plugins.pagination.Page;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.Parameter;
import io.swagger.v3.oas.annotations.tags.Tag;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;

import org.springframework.validation.annotation.Validated;
import org.springframework.web.bind.annotation.*;

import jakarta.validation.constraints.Min;
import jakarta.validation.constraints.NotEmpty;
import java.util.List;

@Slf4j
@RestController
@RequestMapping("/api/conversation")
@RequiredArgsConstructor  // 使用 final 字段注入
@Validated  // 启用方法参数校验
@Tag(name = "会话管理", description = "会话相关接口")
public class ConversationController {

    private final IConversationService conversationService;

    @Operation(summary = "创建会话")
    @PostMapping("/create")
    public Result<ConversationResponse> create(
            @Validated @RequestBody ConversationCreateRequest request) {
        log.info("创建会话: conversationId={}", request.getConversationId());
        return Result.success(conversationService.createConversation(request));
    }

    @Operation(summary = "获取会话详情")
    @GetMapping("/{conversationId}")
    public Result<ConversationResponse> getByConversationId(
            @Parameter(description = "会话ID") @PathVariable String conversationId) {
        return Result.success(conversationService.getByConversationId(conversationId));
    }

    @Operation(summary = "分页查询会话")
    @GetMapping("/page")
    public Result<Page<ConversationResponse>> page(
            @Parameter(description = "当前页") @RequestParam(defaultValue = "1") @Min(1) int current,
            @Parameter(description = "每页数量") @RequestParam(defaultValue = "10") @Min(1) int size,
            @Parameter(description = "搜索关键词") @RequestParam(required = false) String keyword) {
        Page<ConversationResponse> page = conversationService.pageQuery(current, size, keyword);
        return Result.success(page);
    }

    @Operation(summary = "查询所有会话")
    @GetMapping("/list")
    public Result<List<ConversationResponse>> list() {
        return Result.success(conversationService.listAll());
    }

    @Operation(summary = "更新会话")
    @PutMapping("/{id}")
    public Result<ConversationResponse> update(
            @Parameter(description = "会话ID") @PathVariable @Min(1) Long id,
            @Validated @RequestBody ConversationUpdateRequest request) {
        log.info("更新会话: id={}", id);
        return Result.success(conversationService.updateConversation(id, request));
    }

    @Operation(summary = "删除会话")
    @DeleteMapping("/{id}")
    public Result<Void> delete(
            @Parameter(description = "会话ID") @PathVariable @Min(1) Long id) {
        log.info("删除会话: id={}", id);
        conversationService.deleteConversation(id);
        return Result.success();
    }

    @Operation(summary = "批量删除会话")
    @DeleteMapping("/batch")
    public Result<Void> batchDelete(
            @RequestBody @NotEmpty(message = "删除ID列表不能为空") List<@Min(1) Long> ids) {
        log.info("批量删除会话: ids={}", ids);
        conversationService.batchDeleteConversation(ids);
        return Result.success();
    }
}
```

## 五、最佳实践总结

### 1. 异常处理最佳实践

✅ **DO**：
- 使用全局异常处理器统一处理异常
- 区分业务异常和系统异常
- 记录详细的异常日志
- 返回用户友好的错误信息

❌ **DON'T**：
- 不要在 Controller 层捕获异常
- 不要向用户暴露系统内部错误
- 不要忽略异常或只打印日志

### 2. 缓存使用最佳实践

✅ **DO**：
- 为不同类型的数据设置不同的过期时间
- 写操作及时清除相关缓存
- 使用明确的缓存键
- 避免缓存空值

❌ **DON'T**：
- 不要缓存频繁变化的数据
- 不要使用过长的缓存时间
- 不要忘记清除缓存

### 3. 代码设计最佳实践

✅ **DO**：
- 遵循单一职责原则
- 使用清晰的方法命名
- 添加完整的注释和文档
- 使用参数校验

❌ **DON'T**：
- 不要写过长的方法
- 不要使用魔法数字
- 不要忽略边界条件
