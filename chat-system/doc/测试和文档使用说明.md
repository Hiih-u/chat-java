# 测试和文档使用说明

## 一、API 文档（Swagger/Springdoc）

### 1. 访问地址

启动应用后，可以通过以下地址访问 API 文档：

- **Swagger UI 界面**：http://localhost:8081/swagger-ui.html
- **OpenAPI JSON 文档**：http://localhost:8081/v3/api-docs
- **OpenAPI YAML 文档**：http://localhost:8081/v3/api-docs.yaml

### 2. 功能特性

- ✅ 自动生成 RESTful API 文档
- ✅ 在线测试 API 接口
- ✅ 查看请求/响应示例
- ✅ 参数说明和验证规则
- ✅ 支持多种数据格式（JSON、YAML）

### 3. 配置说明

配置文件位于：[`OpenApiConfig.java`](../src/main/java/com/ai/chat/config/OpenApiConfig.java:1)

可以自定义：
- API 标题和描述
- 版本号
- 联系信息
- 许可证信息

### 4. 注解使用

在 Controller 中使用以下注解：

```java
@Tag(name = "模块名称", description = "模块描述")
public class YourController {
    
    @Operation(summary = "接口摘要", description = "接口详细描述")
    @GetMapping("/path")
    public Result<Data> method(
        @Parameter(description = "参数描述") @PathVariable String param) {
        // ...
    }
}
```

## 二、单元测试（JUnit 5 + Mockito）

### 1. 测试结构

```
src/test/java/
├── com/ai/chat/
│   ├── controller/          # Controller 层测试（MockMvc）
│   │   └── ConversationControllerTest.java
│   └── service/             # Service 层测试（Mockito）
│       └── ConversationServiceTest.java
```

### 2. Service 层测试

**测试文件**：[`ConversationServiceTest.java`](../src/test/java/com/ai/chat/service/ConversationServiceTest.java:1)

**特点**：
- 使用 `@ExtendWith(MockitoExtension.class)` 启用 Mockito
- 使用 `@Mock` 模拟依赖
- 使用 `@InjectMocks` 注入被测试对象
- 测试业务逻辑和异常处理

**示例**：
```java
@ExtendWith(MockitoExtension.class)
class ServiceTest {
    @Mock
    private Mapper mapper;
    
    @InjectMocks
    private ServiceImpl service;
    
    @Test
    void testMethod() {
        // Given
        when(mapper.select()).thenReturn(data);
        
        // When
        Result result = service.method();
        
        // Then
        assertNotNull(result);
        verify(mapper, times(1)).select();
    }
}
```

### 3. Controller 层测试

**测试文件**：[`ConversationControllerTest.java`](../src/test/java/com/ai/chat/controller/ConversationControllerTest.java:1)

**特点**：
- 使用 `@WebMvcTest` 只加载 Web 层
- 使用 `MockMvc` 模拟 HTTP 请求
- 使用 `@MockBean` 模拟 Service 层
- 测试请求参数验证和响应格式

**示例**：
```java
@WebMvcTest(YourController.class)
class ControllerTest {
    @Autowired
    private MockMvc mockMvc;
    
    @MockBean
    private YourService service;
    
    @Test
    void testApi() throws Exception {
        mockMvc.perform(get("/api/path")
                .param("key", "value"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.code").value(200));
    }
}
```

### 4. 运行测试

**命令行运行**：
```bash
# 运行所有测试
mvn test

# 运行指定测试类
mvn test -Dtest=ConversationServiceTest

# 运行指定测试方法
mvn test -Dtest=ConversationServiceTest#testCreateConversation_Success
```

**IDE 运行**：
- 右键测试类或方法 → Run Test
- 使用快捷键（IntelliJ IDEA: Ctrl+Shift+F10）

### 5. 测试覆盖率

查看测试覆盖率（需要配置 JaCoCo 插件）：
```bash
mvn clean test jacoco:report
```

报告位置：`target/site/jacoco/index.html`

## 四、最佳实践

### 1. 测试命名规范

- 测试类：`XxxTest`
- 测试方法：`test方法名_场景_预期结果`
- 使用 `@DisplayName` 提供中文描述

### 2. 测试结构（AAA 模式）

```java
@Test
void testMethod() {
    // Arrange（准备）
    // 准备测试数据和 Mock 行为
    
    // Act（执行）
    // 调用被测试方法
    
    // Assert（断言）
    // 验证结果和行为
}
```

### 3. Mock 原则

- Service 层测试：Mock Mapper 层
- Controller 层测试：Mock Service 层
- 只 Mock 直接依赖，不要过度 Mock

### 4. 测试覆盖

每个公共方法至少包含：
- ✅ 正常场景测试
- ✅ 异常场景测试
- ✅ 边界条件测试
- ✅ 参数验证测试

## 五、常见问题

### 1. 测试时找不到 Bean

使用 `@WebMvcTest` 时只会加载 Web 层，需要用 `@MockBean` 模拟其他依赖。

### 2. 数据库连接失败

确保使用测试配置文件 `application-test.yml`，它使用 H2 内存数据库。

### 3. Mock 不生效

检查是否正确使用了 `@Mock` 和 `@InjectMocks` 注解，以及是否添加了 `@ExtendWith(MockitoExtension.class)`。
